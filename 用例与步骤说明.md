# POS UI 自动化用例与步骤说明

本文档汇总当前仓库中已编写的 Playwright 用例：**中文 `describe` / `test` 标题**，以及执行过程中会进入 Allure 等报告的主要步骤（**`@step` 装饰器**与部分 **`test.step`**）。

- **业务步骤**：`flows/` 中类方法的 `@step('业务步骤：…')`。
- **页面步骤**：`pages/` 中方法的 `@step('页面操作：…')` / `@step('页面读取：…')`。
- **测试步骤**：少数用例在 `utils/step` 或 `test.step` 中定义。

若某分支未执行（例如未出现 License 页则跳过授权流程），报告中不会出现对应步骤。

**统计**：Smoke 14 条 + E2E/契约 11 条 = **25 条**（以 `test(...)` 为准）。

## 三、维护说明

- 新增或修改用例时，请同步更新本文档中的**标题**与**步骤**（以 `flows/`、`pages/` 的 `@step` 及 spec 内 `test.step` 为准）。  
- 步骤文案以代码中的 `@step(...)` 字符串为权威来源。  
- **等待机制**：`expect` 默认等待 **30 秒**；`waitUntil` 未传 `timeout` 时默认 **10 秒**（见 `playwright.config.ts`、`utils/wait.ts`）；单条用例总超时由配置项 `timeout` 及必要时 `test.setTimeout` 另行设定。
