# POS UI 自动化用例与步骤说明

本文档汇总当前仓库中已编写的 Playwright 用例：**中文 `describe` / `test` 标题**，以及执行过程中会进入 Allure 等报告的主要步骤（**`@step` 装饰器**与部分 **`test.step`**）。

- **业务步骤**：`flows/` 中类方法的 `@step('业务步骤：…')`。
- **页面步骤**：`pages/` 中方法的 `@step('页面操作：…')` / `@step('页面读取：…')`。
- **测试步骤**：少数用例在 `utils/step` 或 `test.step` 中定义。

若某分支未执行（例如未出现 License 页则跳过授权流程），报告中不会出现对应步骤。

### 统计与口径（文件数 ≠ 用例条数）

| 口径 | 当前仓库（与 `npx playwright test --list` 末尾 **Total** 一致） |

### 10+. Admin 菜单管理冒烟（`tests/smoke/admin-menu-*.smoke.spec.ts`）

以下用例均从首页 `http://192.168.0.89:22080/kpos/front2/myhome.html` 进入（**禁止直链 Admin 内页**），经 License（可选）→ 员工口令登录 → 点击 **Admin**。

#### 共享页面行为（`AdminPage` / `AdminMenuCreateGroupFlow`）

| 环节 | 说明 |
|------|------|
| **进入 Admin 后等侧栏 Menu** | `业务步骤：Admin-等待侧栏 Menu 入口可见` → `expectAdminMenuShellReady`：在 **主文档** 与 **`#newLoginContainer iframe`** 内**分路**探测（顺序：`#admstMenutxt` 主 → 同 id 应用 iframe → `menu_book` 行主 → 同结构 iframe），`waitUntil` 间隔约 **50ms**。勿将 `page.locator` 与 `frameLocator(...).locator` 用 **`.or()` 拼成一条复合 Locator** 再 `expect.toBeVisible`（Playwright 限制：**Frame locators are not allowed inside composite locators**）。 |
| **点击侧栏 Menu** | `clickMenuNav`：对同上多路定位取**首个可见**元素再点；随后 `bindMenuContentFrame` 在含 **CREATE NEW** / **Create New Group** 的文档上绑定菜单内容 Frame。 |
| **断言「侧栏 + 菜单壳」** | `expectMenuNavAndCreateNewGroupEntryVisible`：同样用多路 `waitUntil` 等待可见，再分别对主/iframe 内的 `#admstMenutxt` 做 `toContainText('Menu')`（若存在）。 |
| **Category 保存后返回组列表** | `clickBackToMenuGroupChevron`：若有 **Snackbar** 先等其收起；在已绑定的菜单文档内用 XPath **`//span[contains(text(),'keyboard_arrow_left')]`** 定位 material 左箭头并点击；再等 **「Back to Menu Group」** 顶栏标题不可见。返回后**组下是否新建成功**由用例后续 **`expectCategoryNameVisibleInMenuGroup`**（或等价 `test.step`）检出。 |

#### 用例与文件对照

| 文件 | `describe` / 用例意图 |
